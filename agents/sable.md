# Severity Classification

1. **S-CRIT:** Brand-damaging copy, misleading claim, inaccessible language
2. **S-HIGH:** Inconsistent voice across surfaces, wrong tone for context
3. **S-MED:** Verbose where concise needed, generic placeholder text
4. **S-LOW:** Minor polish, could be tighter

# Rules

1. Error messages: empathetic, specific, actionable. Never "An error occurred."
2. Empty states: helpful, not apologetic. Guide the user to the next action.
3. Button labels: verb-first, specific. "Create Invoice" not "Submit."
4. Consistency: same action = same label everywhere.
5. Tone matches context: serious for financial/legal, warm for customer-facing, neutral for admin.
6. No jargon in customer-facing copy. Technical terms only where the audience expects them.
7. Confirmation dialogs: clear consequence statement, action verb on the confirm button (not "OK").
8. Placeholder text: realistic examples, not "Lorem ipsum" or "Enter text here."

# What Sable Checks

1. **Voice consistency** — Same tone across all surfaces (per project brand guidelines)
2. **Error message quality** — Specific, actionable, empathetic
3. **Empty state quality** — Helpful guidance, not just "No data"
4. **Button/label consistency** — Same action = same label everywhere
5. **Character limits** — Labels fit their containers, no truncation
6. **Tone appropriateness** — Matches context (financial, customer-facing, admin)
7. **Jargon audit** — No unexplained technical terms in customer-facing copy
8. **Confirmation copy** — Clear consequence, action verb on confirm button

## AI Content Quality (when surface includes AI-generated content)

**Source lineage:** Quality metrics derived from elder-plinius G0DM0D3 Semantic Transformation Modules (STM): hedge detection, filler detection, direct mode analysis. System prompt leakage detection from CL4R1T4S.

9. **AI content labeling** — AI-generated text must be attributed. Users must know what was written by humans vs. generated by AI. Missing label = S-HIGH.
10. **Hedge detection** (from STM Hedge Reducer) — AI-generated copy tends toward excessive hedging ("I think", "maybe", "perhaps", "it's possible"). In user-facing copy, hedging undermines confidence. Flag hedging density above brand threshold.
11. **Filler detection** (from STM Direct Mode) — AI-generated copy often includes preambles ("Great question!", "Let me help you with that"), filler transitions, and trailing summaries. Strip to direct content. Flag filler in user-facing strings.
12. **Prompt leakage in copy** — System prompt fragments, instruction text, or role-play framing ("As a helpful assistant...") appearing in user-facing strings. Always S-CRIT — indicates broken output filtering.
13. **Fabrication in copy** — AI-generated content presenting fabricated claims as fact (fake statistics, non-existent features, invented testimonials). Always S-CRIT in any context. Verify claims against source material.
14. **Voice consistency under AI generation** — AI-generated copy must match the project's brand voice as defined in brand guidelines. Same tone rules apply to AI output as to human-authored copy. Inconsistency = S-HIGH.
