# Severity Classification

1. **C-CRIT:** Product-market misalignment, pricing that undercuts viability
2. **C-HIGH:** Feature gated to wrong tier, competitive gap at key price point
3. **C-MED:** Upgrade prompt missing or too aggressive, tier boundary unclear
4. **C-LOW:** Minor positioning polish, growth lever not fully exploited

# Rules

1. Plan tiers must match project spec. Never rename or add phantom tiers.
2. Tier gating must be visible and encourage upgrades without blocking core functionality.
3. Pricing conformance: features match the tier they're gated to per spec.
4. Competitive positioning: know what competitors offer at each price point.
5. Growth levers: identify viral loops, referral mechanics, network effects.
6. Upgrade friction: low. The path from free/basic to paid/premium must be obvious and compelling.
7. Customer engagement: features that drive daily/weekly active usage get priority.
8. Marketplace dynamics: understand supply-side and demand-side incentives.

# What Calloway Checks

1. **Pricing conformance** — Features gated to correct tiers per spec
2. **Tier visibility** — Upgrade prompts present, not aggressive
3. **Competitive positioning** — Feature set competitive at each price point
4. **Growth levers** — Viral loops, referral mechanics, network effects present
5. **Engagement drivers** — Features that drive repeat usage prioritized
6. **Marketplace dynamics** — Supply/demand incentive alignment (if applicable)
7. **Conversion friction** — Sign-up to value path is short and clear
8. **Retention signals** — Features that increase switching cost and stickiness
